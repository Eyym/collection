用户管理和权限管理

2019-04-23
作者：loubth

用户权限管理：为了保证数据库数据的安全，在不同的项目中会给不同的角色（开发者）不同的操作权限

通常，一个用户的密码不会长期不变，所以需要经常性的变更数据库用户密码来确保用户本身安全（mysql客户端用户）

用户管理

我们注意到Host字段和User字段组成了user表的复合主键

创建用户

理论上讲可以采用两种方式创建用户：

  1. 直接使用root用户在mysql.user表中插入记录（不推荐）
  2. 专门创建用户的SQL指令

    基本语法：CREATE USER '用户名'@'主机地址' IDENTIFIED BY '明文密码';

修改用户密码

Mysql中提供了多种修改的方式：以前基本上都必须使用对应提供的一个系统函数：password()，需要靠该函数对密码进行加密处理，从MySQL 5.7.6版本开始，MySQL不推荐使用此函数，在高版本中已经将其删除。作为一个代替的解决方案，它使用明文密码

  1. 使用ALTER USER指令

    基本语法：ALTER USER '用户名'@'主机地址' IDENTIFIED BY '新密码';

    示例

    1
    2
    mysql> ALTER USER 'user1'@'%' IDENTIFIED  BY '1234567';
    Query OK, 0 rows affected (0.04 sec)
  2. 使用专门的修改密码的指令

    旧基本语法：SET PASSWORD FOR '用户名'@'主机地址'=PASSWORD('新密码');      已经弃用

    新基本语法：SET PASSWORD FOR '用户名'@'主机地址'='新密码';

    示例

    1
    2
    mysql> SET PASSWORD FOR 'user1'@'%'='654321';
    Query OK, 0 rows affected (0.02 sec)
  3. 使用更新语句update来修改表

    旧基本语法：UPDATE mysql.user SET password=PASSWORD(新的明文密码) WHERE user='用户名' AND host='主机地址';

    示例（失败）

    1
    2
    3
    mysql> UPDATE mysql.user SET authentication_string='1234567' WHERE user='user1' AND host='%';
    Query OK, 1 row affected (0.07 sec)
    Rows matched: 1  Changed: 1  Warnings: 0

    虽然显示更新成功，但是你用新密码登录会失败的，因为这只是一条普通的更新语句，mysql.user表中密码字段（authentication_string）保存的是加密过的密码。使用这条语句会直接将旧密码改成未加密过的明文新密码，当客户端登录的时候，用户输入密码，数据库系统将其加密与数据库中存储的明文密码比对，结果当然就是匹配失败，导致无法登录

权限管理

在mysql中将权限管理分为三类：

  1. 数据权限：增删改查（select\update\delete\insert）
  2. 结构权限：结构操作（create\drop）
  3. 管理权限：权限管理（create user\grant\revoke）：通常只给管理员

授予权限：grant

将权限分配给指定的用户

基本语法：GRANT 权限列表 ON 数据库.表名 TO 用户;

权限列表：使用逗号分隔，但是可以使用“all privileges”代表全部权限
数据库.表名：可以是单表（数据库名字.表名），可以是具体某个数据库（数据库.*），也可以整库（*.*）
用户：'用户名'@'主机地址'

刷新权限：flush

Flush：刷新，将当前对用户的权限操作，进行一个刷新，将操作的具体内容同步到对应的表中

基本语法：FLUSH PRIVILEGES;

1
2
mysql> FLUSH PRIVILEGES;
Query OK, 0 rows affected (0.02 sec)

那这个命令什么时候该用？可以参考官方文档

当mysqld启动时，所有的权限都会被加载到内存中

如果使用GRANT/REVOKE/SET PASSWORD/RENAME USER命令来更改数据库中的权限表，mysqld服务器将会注意到这些变化并立即加载更新后的权限表至内存中，即权限生效

如果使用INSERT/UPDATE/DELETE语句更新权限表，则内存中的权限表不会感知到数据库中权限的更新，必须重启服务器或者使用FLUSH PRIVILEGES命令使更新的权限表加载到内存中，即权限需在重启服务器或者FLUSH PRIVILEGES之后方可生效

权限生效的含义：

  1. 表级别/列级别的权限，当更新后的权限加载至内存表中，已存在的会话下一次请求时可使用该权限，在修改权限后的建立的会话则立即生效
  2. 数据库级别的权限，当更新后的权限加载至内存表中，已存在会话下一次使用USE db_name后，可使用该权限，在修改权限后的建立的会话则立即生效
  3. 全局权限或者修改密码，当更新后的权限加载至内存表中，需要在下一次登录mysqld后，可使用该权限或密码，对已存在会话不起作用

密码丢失的解决方案

如果忘记了普通用户的密码，使用root用户重新设置即可，如果忘记了root用户密码，就需要去找回或者重置root用户密码，如何重置root用户密码呢？
  1. 停止mysql服务（要使用管理员权限运行cmd）：net stop mysql
