整数类型

mysql有如下几种整数类型

类型（默认有符号）大小范围（有符号）范围（无符号）描述
TINYINT1字节[-128，127][0，255]迷你整型
SMALLINT2字节[-32768，32767)][0，65535]小整型
MEDIUMINT3字节[-8388608，8388607][0，16777215]中整型
INT或INTEGER4字节[-2147483648，2147483647][0，4294967295]标准整型
BIGINT8字节[-9233372036854775808，9223372036854775807][0，18446744073709551615]大整型

创建表的时候如果添加整型字段，默认情况是有符号整型（可以存储负值），如果想要添加无符号的整型字段（不可以存储负值），需要使用unsigned这个字段属性

但这时我们又发现了一个问题，就是字段类型tinyint后面跟着一个“(3)”，这个“(3)”表示什么含义呢？

其实这表示的是“显示长度”，显示长度是指数据（本例中是迷你整型）在显示的时候，最多会占几位（包括符号位）

无符号tinyint类型可表示的整型范围为0～255，又不能表示负数不用考虑负号，因此最多占3位

如果是有符号迷你整型，它的表示范围是-128～127，可以表示负数需要考虑负号，因此最多占4位,tinyint后会跟一个“(4)”

小数类型

mysql中将小数又分为两种类型：浮点型和定点型

浮点型

浮点型又称之为精度类型：是一种有可能丢失精度的数据类型，数据有可能不那么准确（尤其是在超出精度范围的时候）

浮点型之所以能够存储较大的数值（不精确），原因就是利用存储数据的位来存储指数。

类型

数符(s)

阶码(E)

尾数(M)

总位数

有效数字

数值范围

Float

1

8

23

32

6～7

-3.4*10^38  ～  +3.4*10^38

Double

1

11

52

64

15～16

-1.7*10^-308  ～  1.7*10^308

这其中数符来存储正负，阶码来存储指数，尾数存储有效数字

Float

又称之为单精度浮点型：系统提供4个字节用来存储数据，但是能表示的数据范围比整型大的多，大概是10^38，只能保证大概7个左右的精度（如果数据在7位数以内，那么基本是准确的，但是如果超过7位数，那么就是不准确的）

基本语法
Float：表示不指定小数位的浮点数
Float(M,D)：表示一共存储M个有效数字，其中小数部分占D位，例如Float(10,2)表示整数部分为8位，小数部分为2位

咦？我们明明存入的是12345678.90，怎么查看的时候变成了12345679.00？

仔细思考一下便明白，前面已经提到过float类型的数据有效数字（代表精确度）是6～7位，也就是说从第7位开始就不一定准确了，所以才产生了上面的结果

注意：如果数据精度丢失，那么浮点型是按照四舍五入的方式进行计算的

我们来尝试插入一个超过设定位数的数据观察会有什么结果

1
2
mysql> INSERT INTO my_float VALUES(123.1234567,123456789.00);
ERROR 1264 (22003): Out of range value for column 'f2' at row 1

果然系统提示插入数据超出范围，因为我们创建表时指定第二个字段是float(10,2)，意味着整数部分不能插入超过8位的数字

我们在来尝试插入一个特殊的数字，这个特殊的数字长度刚好满足条件，但是会超出精度，例如我们插入“99999999.99”，这个数字整数部分8位，小数部分2位，刚好满足条件，但是float类型的精度不够8位，此时有效数字后面的数字会采用“四舍五入”的原则进行舍入，舍入后数字整数部分变为9位，不再满足条件，我们来观察一下结果

浮点数的应用：通常是用来保存一些数量特别大，大到可以不用那么精确的数据

Double

Double又称之为双精度浮点型：系统用8个字节来存储数据，表示的范围更大，10^308次方，但是精度也只有15位左右。其它特性可以参考float

定点数

定点数：能够保证数据精确的小数（小数部分可能不精确，超出长度会四舍五入），整数部分一定精确

Decimal

Decimal定点数：系统自动根据存储的数据来分配存储空间，每大概9个数就会分配四个字节来进行存储，同时小数和整数部分是分开的

基本语法
Decimal(M,D)：M表示总长度，最大值不能超过65，D代表小数部分长度，最长不能超过30

Date

日期类型：系统使用3个字节来存储数据，对应的格式为：YYYY-mm-dd，能表示的范围是从1000-01-01～9999-12-31，初始值为
0000-00-00

Time

时间类型：能够表示某个指定的时间，系统提供3个字节来存储，对应的格式为：HH:mm:ss，但是mysql中的time类型能够表示时间范围要大的多，能表示从-838:59:59～838:59:59，在mysql中具体的用处是用来描述时间段

Datetime

日期时间类型：就是将前面的date和time合并起来，表示的时间，使用8个字节存储数据，格式为YYYY-mm-dd HH:mm:ss，能表示的区间1000-01-01 00:00:00到9999-12-31 23:59:59，其可以为0值：0000-00-00 00:00:00

Timestamp

时间戳类型：时间戳是指格林威治时间1970年01月01日00时00分00秒(北京时间1970年01月01日08时00分00秒)起至现在的总秒数，使用4个字节存储数据，mysql中的时间戳只是表示从格林威治时间开始，但是其格式依然是YYYY-mm-dd HH:mm:ss

Year

年类型：占用1个字节来保存，能表示1901～2155年，但是year有两种数据插入方式：0～99和四位数的具体年

字符串类型

Char

定长字符串：指定长度之后，系统一定会分配指定的空间用于存储数据

基本语法：char(L)，L代表字符数（中文与英文字母一样），L长度为0到255

Varchar

变长字符串：指定长度之后，系统会根据实际存储的数据来计算长度，分配合适的长度（数据不能超出指定长度）

基本语法：varchar（L），L代表字符数，L的长度理论值位0到65535因为varchar要记录数据长度（系统根据数据长度自动分配空间），所以每个varchar数据产生后，系统都会在数据后面增加1-2个字节的额外开销，是用来保存数据所占用的空间长度，如果数据本身小于127个字符，额外开销一个字节，如果大于127个，就开销两个字节

注意:char和varchar的区别绝对不是char(L)类型只能存L个字符，事实上char(L)类型与varchar(L)一样也能存小于L个字符，但是即使如此，char(L)依然会占用L个字符所占的空间，这就是两者的区别

char和varchar数据存储对比（假设使用utf8，一个字符都会占用3个字节）

存储数据char(2)varchar(2)char所占字节varchar所占字节
AAA2*3=61*3+1=4
ABABAB2*3=62*3+1=7

这就体现出了二者的区别，char(L)所占空间是确定的，varchar(L)会随着所存字符数改变而改变

另外char有一个优点就是char的查询效率要比varchar高

使用场景：char和varchar都是在字符数量不是很多的时候使用，如果确定数据一定是占指定长度，那么使用char类型，如果不确定数据到底有多少，那么使用varchar类型。如果数据长度超过255个字符，不论是否固定长度，都会使用text，不再使用char和varchar

Text和Blob

text用来存储普通的字符文本，blob（binary large object）存储一个二进制大对象，mysql系统提供四种text和blob

TinyBlob最大长度255个字元(2^8-1)
TinyText最大长度255个字元(2^8-1)
Blob最大长度65535个字元(2^16-1)
Text最大长度65535个字元(2^16-1)
MediumBlob最大长度16777215个字元(2^24-1)
MediumText最大长度16777215个字元(2^24-1)
LongBlob最大长度4294967295个字元 (2^32-1)
LongText最大长度4294967295个字元 (2^32-1)

blob和text类型的区别

  1. blob列被视为二进制字符串(字节字符串)
  2. text列被视为非二进制字符串(字符字符串)
  3. blob列没有字符集，并且排序和比较基于列值字节的数值值
  4. text列有一个字符集，并且根据字符集的 校对规则对值进行排序和比较

blob和text类型的相同点：在text或blob列的存储或检索过程中，不存在大小写转换

Enum

枚举类型：在数据插入之前，先设定几个项，这几个项就是可能最终出现的数据结果

如果确定某个字段的数据只有那么几个值：如性别，男、女、保密，系统就可以在设定字段的时候规定当前字段只能存放固定的几个值：使用枚举

基本语法：enum(数据值1,数据值2,...)

系统提供了1到2个字节来存储枚举数据：通过计算enum列举的具体值来选择实际的存储空间：如果数据值列表在255个以内，那么一个字节就够，如果超过255但是小于65535，那么系统采用两个字节保存

Set

集合：是一种将多个数据选项可以同时保存的数据类型，本质是将指定的项按照对应的二进制位来进行控制，1表示该选项被选中，0表示该选项没有被选中。

基本语法：set('值1','值2','值3',...)

系统为set提供了多个字节进行保存，但是系统会自动计算来选择具体的存储单元

1个字节：set只能有8个选项
2个字节：set只能有16个选项
3个字节：set只能有24个选项
4个字节：set只能有32个选项
8个字节（最多）：set只能有64个选项

set和enum一样，最终存储到数据字段中的依然是数字而不是真实的字符串

总结：enum类似单选，而set类似多选

所有字符串类型数据要注意：在mysql中，有一项规定：mysql的记录长度（record也可以理解成行）总长度不能超过65535个字节，所以在mysql中如果你发现varchar等类型数据明明没有达到本数据类型规定的极限但却被禁止插入就是这个原因了
