直接插入排序

2019-06-28

思路

基本思想

直接插入排序（Insertion Sorting）的基本思想是：把n个待排序的元素看成为一个有序表和一个无序表，开始时有序表中只包含一个元素，无序表中包含有n-1个元素，排序过程中每次从无序表中取出第一个元素，把它的排序码依次与有序表元素的排序码进行比较，将它插入到有序表中的适当位置，使之成为新的有序表

过程

public static void insertSort(int[] arr) {
    //定义两个临时变量，一个用来放要插入的值，一个用来放要插入位置的下标
    int insertVal = 0;
    int insertIndex = 0;
 
    for (int i = 1; i < arr.length; i++) {  //arr[0]插到自己的位置没有意义，所以从1开始
        //初始化临时变量
        insertVal = arr[i];     //arr[i]代表要寻找插入位置的数字的值
        insertIndex = i - 1;    //假定插入的位置就在紧挨着arr[i]的前面
 
        //寻找真正的要插入的位置（从后往前）
        while (insertIndex >= 0 && insertVal < arr[insertIndex]) {  //判断索引是否越界以及比较待插入的值与当前索引所指的元素的值的关系
            arr[insertIndex + 1] = arr[insertIndex];    //把当前索引所指的元素向后挪一个位置
            insertIndex--;  //当前索引向前指一个位置
        }
 
        //当退出while循环时，表示此时待插入的值insertVal已经不小于当前索引所指的元素的值arr[insertIndex]了。显然我们找到了要插入的位置就是arr[insertIndex]后面的位置arr[insertIndex+1]
        if (insertIndex + 1 != i) {     //判断待插入元素的位置与要插入的位置是否相同，也就是说待插入的值是否原本就在正确的位置
            arr[insertIndex + 1] = insertVal;   //把待插入的值插入我们找到的位置
        }
 
    }
 
}

优化

我们发现当要插入的数是较小的数时，前面的数后移的次数明显增多，对效率有影响。如果想解决这个问题，我们可以使用希尔排序
