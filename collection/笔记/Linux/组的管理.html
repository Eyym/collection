用户组介绍

类似于角色，系统可以对有共性的多个用户进行统一的管理

Linux的用户需要至少要属于一个“组”，不能独立于组外，创建用户时如果不为其指定一个组，那么系统会自动为其创建一个同名的组，并将这个用户添加到这个组内。

Linux中有两种类型的组。每个用户应该只有一个主要组（主组）primary group和任意数量的次要组（附加组）secondary group

主要组（主组）：创建用户帐户时，已将主要组添加到用户。它通常是用户的名称。在执行诸如创建新文件（或目录）、修改文件或执行命令等任何操作时，主要组将应用于用户。用户的主要组信息存储在/etc/passwd文件中

次要组（附加组）：它被称为次要组。它允许用户组在同一组成员文件中执行特定操作。例如，如果你希望允许少数用户运行Apache（httpd）服务命令，那么它将非常适合

增加组

基本语法

基本语法：groupadd 组名

示例（添加一个名为wudang的组）

1
[root@host-loubth ~]# groupadd wudang

删除组

基本语法

基本语法：groupdel 组名

示例

1
[root@host-loubth ~]# groupdel wudang

创建用户时直接加上组

基本语法

基本语法：useradd -g 用户组 用户名

示例（增加一个用户zwj，直接将他指定到wudang组）

1
2
3
4
[root@host-loubth ~]# groupadd wudang
[root@host-loubth ~]# useradd -g wudang zwj
[root@host-loubth ~]# id zwj
uid=501(zwj) gid=502(wudang) 组=502(wudang)

修改用户的主组

基本语法

基本语法：usermod -g 用户组 用户名

示例（创建一个shaolin组，将zwj用户修改到shaolin组）

1
2
3
4
5
[root@host-loubth ~]# id zwj
uid=501(zwj) gid=502(wudang) 组=502(wudang)
[root@host-loubth ~]# usermod -g shaolin zwj
[root@host-loubth ~]# id zwj
uid=501(zwj) gid=503(shaolin) 组=503(shaolin)

配置文件

用户（user）的配置文件

“/etc/passwd”文件，记录用户的各种信息，每行的含义：
用户名:口令:用户标识号:主组标识号:注释性描述:主目录（家目录）:登录Shell

口令（密码）已加密，具体存放在口令配置文件里，这里用x代替
