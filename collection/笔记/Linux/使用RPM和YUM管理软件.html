rpm包的管理

介绍

rpm是RedHat Package Manager（RedHat软件包管理工具）的缩写，类似windows的setup.exe。rpm原本是Red Hat Linux发行版专门用来管理Linux各项套件的程序，由于它遵循GPL规则且功能强大方便，因而广受欢迎。逐渐受到其他发行版的采用。rpm套件管理方式的出现，让Linux易于安装，升级，间接提升了Linux的适用度。它生成具有.rpm扩展名的文件，这一文件格式名称虽然打上了RedHat的标志，但理念是通用的

rpm基本语法

基本语法：rpm [选项] 软件包

常用选项

  • -a：查询所有软件
  • -e 软件包 或 --erase 软件包：删除指定的软件
  • -f 文件：查询拥有指定文件的软件
  • -h或--hash：套件安装时列出标记
  • -i 软件包 或 --install 软件包：安装指定的软件包
  • -i：显示套件的相关信息
  • -l：显示软件的文件列表
  • --nodeps：不验证包依赖关系
  • -q：使用询问模式，当遇到任何问题时，rpm命令会先询问用户
  • -v：显示命令执行过程

提示：erase语法为 [ɪˈreɪs]，翻译为“擦除”

rpm包名基本格式

示例（firefox）

一个rpm包名：firefox-45.0.1-1.el6.centos.x86_64.rpm

  • 名称：firefox
  • 版本号：45.0.1-1
  • 适用操作系统：el6.centos.x86_64
  • 系统：el6.centos.x86_64表示centos6.x的64位，系统如果是i686、i386表示32位系统，noarch表示通用

卸载rpm包

基本语法

基本语法：rpm -e 软件名称

示例（卸载firefox）

1
2
3
4
5
[root@host-loubth Packages]# rpm -q firefox
firefox-45.0.1-1.el6.centos.x86_64
[root@host-loubth Packages]# rpm -e firefox
[root@host-loubth Packages]# rpm -q firefox
package firefox is not installed

细节问题

如果其它软件包依赖于您要卸载的软件包，卸载时则会产生错误信息

1
2
[root@host-loubth Packages]# rpm -e foo
removing these packages would break dependencies:foo is needed by bar-1.0-1

如果我们就是要删除foo这个rpm包，可以增加参数--nodeps，就可以强制删除，但是一般不推荐这样做，因为依赖于该软件包的程序可能无法运行

“rpm -e --nodeps foo”带上--nodeps就是强制删除

基本语法

基本语法：rpm -ivh rpm包全路径名称

参数说明

  • i=install 安装
  • v=verbose 提示
  • h=hash 进度条

yum包的管理

介绍

Yum（全称为 Yellow dog Updater, Modified）是一个在Fedora和RedHat以及CentOS中的Shell前端软件包管理器。基于RPM包管理，能够从指定的服务器自动下载RPM包并且安装，可以自动处理依赖性关系，并且一次安装所有依赖的软件包，无须繁琐地一次次下载、安装。使用yum的前提是可以联网

基本语法

基本语法：yum [选项] 参数

常用选项

  • -y：对所有的提问都回答“yes”

yum参数

  • install： 安装rpm软件包
  • update： 更新rpm软件包
  • check-update： 检查是否有可用的更新rpm软件包
  • remove： 删除指定的rpm软件包
  • list： 显示软件包的信息
  • search： 检查软件包的信息
  • info： 显示指定的rpm软件包的描述信息和概要信息
  • clean： 清理yum过期的缓存
  • shell： 进入yum的shell提示符
  • resolvedep： 显示rpm软件包的依赖关系
  • localinstall： 安装本地的rpm软件包
  • localupdate： 显示本地rpm软件包进行更新
  • deplist： 显示rpm软件包的所有依赖关系
